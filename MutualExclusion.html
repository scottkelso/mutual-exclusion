CSC321 Concurrent Programming Practical

Mutual Exclusion

The aim of this practical is to gain experience of the problems associated with 'bare machine' mutual exclusion algorithms. It is important that you study and experiment with each program.
  1. The program Attempt1.java is the first attempt at a mutal exclusion solution. This solution has all of the desired properties except for correct behaviour in the absence of contention.
    • Run the program and observe that there is no interleaving of instructions in critical sections.
    • Remove the comments in the methods and run the program again. Note that some instructions (which ones ?) from one process can be interleaved with instructions in the other process's critical section.
    • Call the killMe method from thread1's nonCritcal section. Observe the starvation of thread2.
  2. The program Attempt2.java is the second attempt at a mutual exclusion solution.
    • Run the program and observe that it does not prevent two processes entering critical sections at the same time.
  3. The program Attempt3.java is the third attempt at a mutal exclusion solution.This algorithm can deadlock.
    • Run the program and observe that mutual exclusion is satisified.
    • Remove the comments. The yield instruction should force interleaving in the preProtocols resulting in deadlock.
  4. The program Attempt4.java is the fourth attempt at a mutal exclusion solution.This algorithm can livelock.
    • Run the program and observe livelock.
    • Remove the comments in the preProtocol loop. Run the program and observe that mutual exclusion is satisfied.
  5. Write a program to solve the mutual exclusion algorithm for two processes using Dekker's algorithm. As with the previous four attempts, insert print statements throughout, and experiment with the program.
